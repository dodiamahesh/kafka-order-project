🗂️ Kafka Topics
order-placed
3 partitions Producer
Fired when order is saved. Consumer picks it up to begin processing.
order-processed
3 partitions 2 consumers
Fired after processing. Notification group logs it (simulates email).
📚 Key Kafka Concepts
Topic Named channel — like a table. Messages go in, consumers read them.
Partition Topics split into 3 parts. Each consumer thread handles one partition in parallel.
Consumer Group Group of consumers sharing load. Different groups = independent processing.
Offset Message position in a partition. Kafka remembers where each group stopped.
Producer Acks acks=all → Kafka confirms all replicas received it before acknowledging.
